# 项目路线图

状态：**阶段门驱动**<br>
最后更新：2026-07-13

路线图描述顺序和退出条件，不承诺未经验证的发布日期。时间范围按单人业余开发估算，在完成第一个可玩构建并测得实际速度后修订。

## 原则

- 先验证情感命题，再扩大内容和架构。
- 每个阶段必须产生可玩的端到端版本，而不是孤立系统。
- 未通过阶段门时优先修正线索、节奏和表达，不用新增功能掩盖问题。
- 手工编排默认优先；只有重复劳动成为已观察瓶颈时才建设通用工具。

## M0：制作基线

状态：**完成**

产物：

- 无剧透公开 README；
- 愿景、决策、切片、叙事、技术和测试权威文档；
- 贡献、资产许可和历史归档规则；
- 32px V2 运行时图集、可编辑源和确定性构图预览（状态为 REVIEW）；
- Godot 4.6.3 项目、Mac 开发环境和 Windows x86_64 导出链已初始化并通过 F-01 冒烟测试。

退出条件：所有阻断性产品决定有唯一答案，旧稿不再与当前规格竞争。

## M1：灰盒双轮流程

状态：**进行中（F-01 至 F-04、G-01 至 G-04、N-01 完成；正在进入 N-02/N-03）**

预计：2–4 个集中开发日

范围：

- 使用已锁定的 Godot 4.6.3-stable 初始化项目和导出模板；
- 五房间灰盒、移动、碰撞、交互和文本；
- 三个占位碎片、任意顺序、状态迁移；
- 无最终美术批准要求的惩罚时间线；
- 第二轮变化开关、时钟门槛和两个占位结尾；
- 开发调试入口。

退出条件：从标题到两个结尾都能运行，`PLAYTEST.md` 的状态类 P0 验收通过。

本阶段可以导入现有 V2 资产检查比例和碰撞，但不进行全房间最终美术批准。禁止加入：程序化变异、跨会话存档、完整知识图谱。

## M2：可盲测垂直切片

预计：3–7 个集中开发日

范围：

- 实装 `NARRATIVE_BEATS.md` 的最终文本；
- 低保真但风格一致的像素环境和角色轮廓；
- 完整环境音、心跳、静默和混音安全处理；
- 第二轮四项固定变化和三种行为回应；
- 内容提示、音量、全屏和降低闪烁；
- Windows 发布构建。

退出条件：

- 所有 P0 与 P1 功能验收通过；剩余 P2 有明确记录且不影响核心理解；
- 至少五名不知情测试者完成；
- `PLAYTEST.md` 核心体验阶段门全部通过。

如果未通过：保持 M2，进行一次有假设的修改和一批新测试；不得直接进入 M3。

## M3：叙事 Alpha

预计：在 M2 通过后 3–6 周业余开发

范围：

- 扩展为三轮、25–35 分钟可玩体验；
- 8–10 个主要记忆节点；
- 建立苏岚和秦禾的日常、离开计划及主角暴力模式；
- 第二个和第三个真正的玩家知识门槛；
- 数据驱动的命名变化集和内容校验工具；
- 最小进度保存，仅在完整体验长度证明需要后加入；
- 至少两轮外部测试。

阶段门：玩家能区分“房子的规则”“主角记忆”“受害者事实”，并持续把责任归于主角；循环重复未造成大面积无聊或迷路。

## M4：内容完整 Beta

预计：M3 通过后 6–10 周业余开发

范围：

- 约五轮、45–75 分钟；
- 12–16 个主要记忆节点；
- 揭示杀害、纵火、自焚与惩罚结构；
- 完整面对/回避情感路径，两者都通向轮回继续；
- 全部环境、美术、UI、音频和文本达到内容完整；
- 保存、设置、重置和本地数据说明完成；
- 简体中文完整校对。

可评估但不承诺：受约束的本地行为回应。若实现，必须只存于本机、可清除、不联网，并在设置或隐私说明中如实披露。不得为了宣传台词虚构“已观察玩家几十次”的事实。

阶段门：内容锁定，所有路径可完成，没有叙事 P0/P1，至少两批目标受众测试完成。

## M5：发布候选

预计：M4 通过后 3–5 周业余开发

范围：

- 性能、兼容性、输入、字体和音频回归；
- Windows 干净环境安装与运行验证；
- 许可证和资产清单审计；
- itch.io 无剧透商店页、截图、预告片与内容提示；
- 外部校对和最终盲测；
- 崩溃恢复、保存迁移与版本号规范。

发布门：

- P0/P1 为零；
- 所有分发资产许可清楚；
- 新玩家可在无开发者协助下完成；
- 商店文案不泄露核心身份反转；
- 发布包与仓库对应提交可追溯。

## 暂不排期的方向

- 英文本地化和其他语言；
- macOS、Linux 或主机平台；
- 手柄完整支持；
- 受约束的程序化变化；
- 跨会话个性化回应；
- Steam 发布。

这些方向只有在完整版核心体验稳定、维护成本可承担且不削弱叙事控制时才进入决策记录。
