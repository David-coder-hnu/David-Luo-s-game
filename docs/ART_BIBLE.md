# 美术圣经

状态：**垂直切片视觉基线已锁定**<br>
适用版本：Vertical Slice 0.1<br>
最后更新：2026-07-13

## 1. 视觉命题

> 这不是一栋闹鬼的房子，而是一栋拒绝继续替主人隐瞒的房子。

![《地狱轮回》视觉语言](../assets/branding/visual-language.svg)

视觉设计应先让住宅可信，再让记忆逐步污染可信感。玩家第一次看到每个房间时，应能理解它曾经属于一个普通家庭；第二次看到时，变化不是“多了一个恐怖素材”，而是同一件日常物品开始指控主角。

## 2. 四条视觉原则

### 熟悉先于恐怖

第一轮 80% 的画面是可以解释的日常空间。墙纸、碗柜、儿童床、照片和时钟都要有明确用途。禁止在初始场景堆满血迹、符号和骷髅。

### 缺席比怪物更重要

画面中只有主角拥有完整可移动轮廓。妻女通过生活痕迹、留白、物品高度和构图位置存在。执行者没有精灵、影子或可识别面孔。

### 变化必须可回忆

变化前的物品需要有清楚轮廓和稳定位置，玩家才可能在没有提示时注意到变化。关键物品不得藏进高噪声纹理。

### 火只在记忆边缘出现

焦痕、暖色和灰烬是后期语义，不是全局滤镜。第一轮只在照片、收据边缘和极少数材质出现；惩罚阶段才让焦红侵入画面边缘。

## 3. 核心色板

| 名称 | HEX | 用途 | 限制 |
|---|---|---|---|
| 夜墨 | `#0B0C14` | 背景、最深阴影 | 不吞没可行走边界 |
| 冷墙 | `#242936` | 墙体、家具暗面 | 第一轮主色 |
| 雾青 | `#40525A` | 冷光、金属、远处层次 | 不用于危险提示 |
| 旧纸 | `#C8BFAE` | 文本、纸张、皮肤高光 | 避免纯白 |
| 病灯 | `#C29A5B` | 灯光、记忆吸引点 | 单屏面积不超过 12% |
| 焦红 | `#8D3035` | 惩罚、烧痕、确认危险 | 第一轮单屏面积不超过 3% |
| 灰烬 | `#69656A` | 褪色物体、第二轮残留 | 不与文本同亮度 |
| 死黑 | `#030407` | 惩罚收束、完整黑屏 | 只用于事件和画面外 |

不使用纯白 `#FFFFFF` 或纯红 `#FF0000`。UI 文本默认使用旧纸色，危险反馈用焦红加形状变化，不能只依赖颜色。

## 4. 像素与分辨率

- 内部分辨率：640×360。
- 基础瓦片：16×16 像素。
- 主角占位：16×24 像素，碰撞脚印约 10×8 像素。
- 家具以 16 像素网格对齐；允许装饰层偏移 1–3 像素，但碰撞不得偏离网格。
- 最近邻采样；禁止纹理过滤、亚像素相机运动和非整数世界缩放。
- 主角行走动画：四方向，每方向 4 帧，8–10 FPS。
- 环境循环动画最多 4 帧；静止优先，移动必须具有叙事意义。
- 轮廓线使用同色相深色，不使用统一纯黑描边。

## 5. 构图层级

画面按以下顺序建立注意力：

1. 可行走空间和出口；
2. 当前房间的生活功能；
3. 可调查物的独特轮廓；
4. 叙事异常；
5. 装饰纹理。

如果装饰让玩家看不见门、碰撞边缘或主要碎片，应删除装饰，而不是增加发光描边。

## 6. 房间视觉身份

### 卧室：被清空的自我

- 最低细节密度；冷墙与夜墨为主。
- 单人枕头、空衣架、没有个人照片。
- 第一轮是相对安全的负空间；第二轮只改变被褥温度表达和极轻微床沿压痕。

### 厨房：选择留下的痕迹

- 冷金属、旧瓷砖和病灯构成最硬的材质对比。
- 收据位于低处暖色焦点，碎玻璃只提供轮廓，不反射跳动高光。
- 第二轮两只干净杯子形成明确的成对构图，替代第一轮的孤立污迹。

### 儿童房：生活仍然在场

- 色彩比其他房间多一档，但全部褪色；不要使用“恐怖儿童房”常见的满墙涂鸦。
- 床、身高线、袜子和画具表明秦禾的习惯。
- 儿童画采用比环境高一档的旧纸色，第一轮藏而可见，第二轮主动朝向玩家。

### 客厅：家庭公开形象

- 最规整、最像“正常家庭”的空间。
- 结婚照和时钟位于不同视线轴，避免玩家一次交互同时发现两个谜题。
- 第二轮男人轮廓恢复时，不画五官，只让身高、肩型和服装色块匹配玩家。

### 走廊：执行者的距离

- 细长、低装饰、灯具节奏清楚。
- 惩罚时间线通过逐盏熄灯改变空间深度。
- 禁止在尽头直接放置人影；压迫来自可见距离缩短。

## 7. 角色规则

### 秦峥

- 轮廓略向前收，肩部不对称，表现长期疲惫而非“酷炫反英雄”。
- 第一轮脸部只有两级明暗，不表现明确情绪。
- 服装使用冷墙、灰烬和一个病灯色小面积识别点。
- 不携带武器，不使用血污装饰。

### 苏岚与秦禾

- 切片不制作可移动幽灵精灵。
- 照片、儿童画和物品中的形象保持一致：苏岚偏暖灰，秦禾保留小面积雾青。
- 不使用空洞眼睛、扭曲笑脸、断肢或突然贴脸。

### 执行者

- 没有角色资产。
- 不通过烟雾、人形阴影或眼睛暗示轮廓。
- 仅允许通过光源消失、空间压缩、材质变暗和非空间音频体现。

## 8. 变化强度

| 等级 | 用途 | 允许变化 | 禁止变化 |
|---|---|---|---|
| V0 日常 | 第一轮初见 | 磨损、轻微焦边 | 超自然移动、红光 |
| V1 记忆 | 碎片完成 | 底噪减少、局部饱和度轻降 | 整屏滤镜跳变 |
| V2 回应 | 第二轮 | 物品替换、位置翻转、轮廓恢复 | 随机家具布局 |
| V3 惩罚 | 固定事件 | 灯光熄灭、边缘焦红、视野收束 | 可见怪物、快速闪屏 |

同一物体一次轮回只承担一个主要变化。不要同时移动、换色、破损和播放动画，否则玩家无法理解变化的含义。

## 9. 光照与后处理

- 第一轮整体饱和度基线为 100%，但资产本身使用低饱和色板。
- 每完成碎片，后处理饱和度最多下降 4%，累计不超过 12%。
- 惩罚在三秒内平滑降至约 55%，不是旧稿中的 30%，以保留像素层级和可行走性。
- 第二轮恢复至约 82%，让玩家感到颜色没有完全回来。
- 暗角只在惩罚阶段出现；最大遮挡不得覆盖角色脚下和最近出口。
- 焦红噪点固定在屏幕边缘，低频缓慢变化；“降低闪烁”模式改为静态渐变。

## 10. UI 美术关系

- UI 像是玩家意识中的字幕，不像游戏菜单皮肤。
- 文本框使用半透明夜墨底、1 像素冷墙边线和旧纸文字。
- 主要交互提示不使用发光按钮；按住确认使用逐渐闭合的细环。
- 标题字可以出现轻微烧蚀缺口，但正文像素字体必须保持完整可读。
- 详细布局和状态见 `UI_UX_SPEC.md`。

## 11. 资产完成标准

每个像素资产必须同时提交：

- 原始可编辑文件；
- 导出 PNG；
- 尺寸、锚点、碰撞建议和色板说明；
- 第一轮与第二轮变体（若有）；
- 在 640×360 实机场景中的截图；
- `ASSET_CREDITS.md` 登记。

验收时在 100%、200% 和全屏整数缩放下检查。若资产只有放大后才看得清，说明轮廓失败。

## 12. AI 辅助美术规则

- 生成式图像只用作构图、光影和材质参考，不直接把未经像素重绘的高分辨率图塞进游戏。
- 每次生成提示必须引用本文件的色板、视角、房间用途和禁用元素。
- 任何生成结果都要经过人工像素重绘、网格对齐、色板收敛和许可登记。
- 禁止用“某位在世艺术家的风格”作为提示；使用可描述的媒介、年代、构图和材质语言。
- GitHub 主视觉可以使用高精度像素概念图，但必须明确它是视觉目标，不冒充游戏实机截图。
