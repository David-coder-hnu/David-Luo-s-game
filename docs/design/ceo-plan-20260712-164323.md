## 战略决定：范围扩展 — 做完整游戏

周末原型验证成功后，目标是发布一款完整的 2D 像素心理恐怖游戏。目标平台：itch.io。目标受众：喜欢 Ib/魔女之家/狂父 的像素恐怖核心玩家。

## 为什么这个野心合理

1. **核心循环已经够强。** "进度即陷阱"是一个真正原创的设计论纲，不是换皮。itch.io 的恐怖游戏社区天然奖励原创性。
2. **技术栈选择正确。** Godot 4.x 从原型到完整发布不需要换引擎。程序化系统可以渐进搭建。
3. **一人团队可行的范围。** 2D 像素 + 单一房子场景 + 知识驱动叙事 = 不需要大型团队或预算。时间是你唯一的成本。

## 从原型到完整的路线图

### Phase 1: 原型验证 (Weekend)
- 一个房子、一个轮回、一次重置
- 手摆变体，无程序化
- 目标：证明核心循环的情感冲击力
- **通过标准：你自己关灯戴耳机玩完，感觉'这个循环是对的'**

### Phase 2: 系统搭建 (2-4 weeks)
- 知识图谱系统（记忆碎片依赖关系）
- 轮回管理器（场景状态重置 + 知识状态保留）
- 程序化变异规则 v1（物品偏移、色调变化、照片细节）
- 地狱之神接近度系统（隐藏的'他来了'计量器）
- **通过标准：3 个轮回可以跑通，每轮变化有感知**

### Phase 3: 完整叙事 (4-8 weeks)
- 完整记忆碎片网络（12-20 个碎片覆盖男主犯罪的完整真相）
- 多轮回叙事弧线
  - 轮回 1-2：碎片化闪回，建立不安
  - 轮回 3-4：真相开始拼合，玩家开始抗拒
  - 轮回 5+：面对完整的罪行，地狱之神逼近
- 程序化变异 v2（房间布局微调、事件触发顺序变化）
- 完整音效设计（环境音、空间音频、策略性沉默）
- **通过标准：有人玩完后说'我不知道该不该继续——但停不下来'**

### Phase 4: 打磨发布 (4-6 weeks)
- 像素美术资产完整化（角色、环境、UI）
- 性能优化 + 多平台导出
- 测试：找 3-5 个不认识的人关灯戴耳机玩，观察反应
- itch.io 商店页面 + 预告片
- 发布

## 差异化点 — 为什么这个游戏会脱颖而出

| 维度 | 常规像素恐怖游戏 | 你的游戏 |
|------|------------------|----------|
| 进度 | 越好越强 | 越接近真相越痛苦 |
| 结局 | 逃离/打破循环 | 循环永恒，没有出路 |
| 知识 | 知识=力量 | 知识=惩罚的前奏 |
| 重复 | 玩家记住地图 | 地图变化，不能依赖记忆 |
| 恐怖来源 | 外部威胁（怪物/鬼） | 内部真相（你自己的罪行） |

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| 纯知识驱动缺乏"奖励感" | 每个碎片揭示的内容本身就是钩子——参考 Outer Wilds，好奇心是终极动力 |
| 循环疲劳 | 每轮 ≤15 分钟，叙事内跳过机制（参考 Lily's Well），每轮必须带来新信息 |
| 程序化变异影响叙事连贯性 | 关键叙事物品/房间不变异——只有'环境'物品变化。玩家需要能信赖核心路径 |
| 一人团队时间投入大 | Phase 1-2 确认方向后再投入 Phase 3+。不要一口气跳到完整叙事 |
| 心理恐怖氛围需要打磨经验 | 从简单开始（低保真像素+环境音），逐步迭代。不要一开始就追求"完美的恐怖" |

## 成功指标

- itch.io 评分 ≥4.5/5
- 有人在评论区说"我玩了之后想了三天"
- 至少一个 YouTuber/主播自发玩并做视频
- 你自己关灯戴耳机玩完整版时，在某一个瞬间感到真正的脊背发凉
