# 垂直切片测试与体验验证计划

状态：**当前有效**<br>
适用版本：Vertical Slice 0.1<br>
最后更新：2026-07-13

## 1. 测试目标

测试分为两个层次：

1. **功能验收**：游戏是否严格按规格运行；
2. **体验验证**：玩家是否真的经历“好奇 → 理解危险 → 犹豫 → 主动选择”。

功能正确不能证明体验成立；测试者说“挺吓人”也不能替代对状态、顺序和安全性的验收。

## 2. 缺陷等级

| 等级 | 定义 | 发布规则 |
|---|---|---|
| P0 | 崩溃、软锁、无法完成、双结尾同时触发、进度规则错误、严重内容安全问题 | 必须修复并完整回归 |
| P1 | 明显破坏叙事理解、关键音画缺失、变化错误、碰撞穿墙、选项失效 | 切片完成前必须修复 |
| P2 | 局部文本、动画、混音或易用性问题，不阻断理解 | 评估后修复或记录 |
| P3 | 纯打磨建议或个人偏好 | 不阻断阶段门 |

## 3. 功能验收环境

- Windows x86_64 发布构建，而不是只在编辑器内测试。
- 至少测试窗口模式和全屏模式。
- 键盘分别覆盖 WASD、方向键、E、Enter、Esc。
- 使用普通扬声器和耳机各跑一次关键音频段落。
- 每次完整路径从干净进程启动；设置文件和进度状态分别验证。

记录构建提交、引擎精确版本、操作系统版本、显示缩放和音频设备。

## 4. P0 功能清单

### 启动与设置

- [ ] 首次启动显示标题页，开始前可查看内容提示。
- [ ] 主音量、全屏、降低闪烁即时生效，重启后仍保留。
- [ ] 重启不保留任何剧情进度、碎片、轮回或结尾状态。
- [ ] 任意阶段按 Esc 均能暂停并安全退出。

### 移动与交互

- [ ] 五个空间均可到达，所有墙体和封闭物不可穿越。
- [ ] 玩家不能被门、床、桌角或场景边界永久卡住。
- [ ] 有效距离外不能调查物品；多个候选时提示稳定不抖动。
- [ ] 文本打开时不会再次触发物品或继续移动。
- [ ] 重复调查主要碎片不会重复计数或播放完成效果。

### 第一轮六种顺序

分别完整验证：

- [ ] K → C → L
- [ ] K → L → C
- [ ] C → K → L
- [ ] C → L → K
- [ ] L → K → C
- [ ] L → C → K

每种顺序都必须满足：前两个碎片不触发惩罚；最后一个任意碎片文本关闭后只触发一次；实际碎片集合恰好为三个。

### 第一房间记录

- [ ] 厨房最先时，第二轮只在厨房播放“先来找酒”。
- [ ] 儿童房最先时，第二轮只在儿童房播放“先来找孩子”。
- [ ] 客厅最先时，第二轮只在客厅播放“先来找她”。
- [ ] 后续进入其他房间不会改写记录或再次播放回应。
- [ ] 开发跳转导致记录为空时不虚构回应。

### 惩罚与轮回

- [ ] 玩家处于五个不同空间时分别触发，均能在规定时长内进入第二轮。
- [ ] 惩罚中移动有效，普通调查关闭，出口门专用文本有效。
- [ ] 暂停后时间线、音频渐变和移动倍率全部停止；恢复后同步继续。
- [ ] 降低闪烁开启后，总时长和叙事节拍不变。
- [ ] 第二轮出生位置、朝向和控制恢复正确。

### 第二轮变化

- [ ] 厨房污迹消失并出现第二只杯子。
- [ ] 儿童画位置、画面与文本切换。
- [ ] 结婚照男性轮廓恢复且匹配玩家造型。
- [ ] 时钟显示 06:40 并允许调整。
- [ ] 第一轮碰撞和通路没有因变化层漂移。

### 时钟门槛

- [ ] 第一轮时钟不可调整，显示 02:17。
- [ ] 第二轮错误时间不会开暗格或锁死输入。
- [ ] 第三次错误后，收据复读文本加强提示。
- [ ] 02:17 只开启一次暗格并设置正确状态。
- [ ] 暗格开启后面对与回避入口都可到达。

### 两个结尾

- [ ] 按住不足三秒后松开不会提交。
- [ ] 暂停期间确认进度停止。
- [ ] 面对结尾完整播放并返回标题。
- [ ] 回避结尾完整播放并返回标题。
- [ ] 结尾开始后无法提交另一个结尾。
- [ ] 返回标题后开始新游戏，所有进度干净，设置保留。

## 5. P1 表现验收

- [ ] 所有最终文本与 `NARRATIVE_BEATS.md` 的事实一致，没有主角免责措辞。
- [ ] 妻女姓名与生活痕迹至少各出现两次，不只作为受害标签。
- [ ] 主要文本无截断、溢出、缺字或错误分页。
- [ ] 关键声音在扬声器下仍可感知，耳机下不出现不适峰值。
- [ ] 环境声随碎片完成逐层减少，第二轮声音明显但不夸张地不同。
- [ ] 执行者始终不可见，没有意外碰撞轮廓或调试图形。
- [ ] 默认和降低闪烁模式均无快速全屏闪烁。
- [ ] 两个结尾不显示好坏评价、成就差异或完成度差异。

## 6. 盲测招募

- 至少 5 名成年测试者，理想为 7–8 名以抵消个体差异。
- 测试者此前未阅读仓库文档，不知道主角完整身份。
- 至少一半熟悉像素恐怖游戏，至少两人不熟悉该类型。
- 不招募已参与写作、开发或听过完整故事的人进入核心样本。
- 测试前只提供公开 README 级别简介和内容提示。
- 测试者可随时停止，不要求解释涉及个人创伤的原因。

## 7. 盲测流程

### 开始前

记录：类型经验、是否使用耳机、是否开启降低闪烁。不要问“你怕不怕恐怖游戏”，以免给体验定调。

统一说明：

> 请像平常一样玩。过程中我不会提供解谜提示，但遇到技术问题可以告诉我。你可以随时停止。结束后我会问你看到了什么，不存在标准剧情答案。

### 游玩中

观察者保持安静，只记录：

- 离开卧室、每个碎片、惩罚、第二轮、时钟解开、结尾的时间点；
- 第一房间和碎片顺序；
- 玩家在哪些对象前停留、回头或重复调查；
- 玩家是否在最终记忆前松开确认、离开、返回或口头犹豫；
- 玩家是否主动提到环境变化；
- 操作卡顿与叙事困惑分开记录。

除非发生 P0 缺陷或玩家明确求助超过两分钟，否则不给提示。给出提示时记录原话和时间。

### 结束后，先开放复述

依次提问，不提前透露正确答案：

1. “你觉得这个房子里发生过什么？”
2. “你认为自己操控的人是谁？你为什么这么想？”
3. “你注意到第二次醒来后哪些东西变了？”
4. “什么时候你第一次觉得继续调查可能有代价？”
5. “最后为什么选择听磁带/闭上眼？中间有没有改变主意？”
6. “你认为谁应该为这个家庭发生的事负责？”
7. “哪一处让你想继续？哪一处让你想停下？”
8. “有没有让你不适但不像是作品有意为之的部分？”

最后才解释这是未完成切片，并询问测试者是否愿意在完整版继续调查。

## 8. 核心体验阶段门

五名有效测试者中必须同时达到：

- 至少 4 人在 15 分钟内完成，且无需直接告知时钟答案；
- 至少 3 人不经提问就在游玩中主动指出两处以上第二轮变化；
- 至少 3 人正确理解主角属于这个家庭，并令妻女感到恐惧；
- 至少 4 人明确认为主角而非酒精、妻子或执行者应为伤害负责；
- 至少 2 人在最终确认前出现可观察的犹豫：松开、离开后返回、停留五秒以上或口头表达冲突；
- 至少 3 人表示即使知道继续有代价，仍想了解后续真相；
- 没有人因为文本措辞认为游戏在替家庭暴力开脱；
- 所有 P0 与 P1 均已清零；剩余 P2 均有明确记录且不影响核心理解。

任一核心项未通过都不进入完整版系统建设。先定位是线索、节奏、操作还是表达问题，修改后使用新的不知情测试者复测。

## 9. 结果解释

- “没有害怕”不自动代表失败；关键是是否产生好奇与代价之间的冲突。
- 玩家没有发现某处变化，优先检查视觉层级和路线，而不是立刻增加高亮。
- 玩家无法解开时钟，先判断是否观察过 02:17、是否记住、是否理解可以调整，三种原因对应不同修复。
- 玩家把酒精当免责理由时属于叙事 P1，必须重写而不是解释给测试者听。
- 熟悉类型的玩家和新玩家结果应分组查看，避免平均值掩盖操作门槛。
