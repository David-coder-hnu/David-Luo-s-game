# 运行时美术评审与质量门

状态：**V1 已拒绝；V2 处于 REVIEW**<br>
适用版本：Vertical Slice 0.1<br>
最后更新：2026-07-13

本文记录运行时视觉资产的真实完成度。它不替代 `ART_BIBLE.md`，而是防止尺寸、色板和图集脚本通过后被误写为美术批准。

## 1. V1 拒绝原因

- 16×16 环境在 640×360 中无法提供目标住宅的材质和房间身份；
- 道具更接近功能图标，透视、比例和生活痕迹不足；
- 主角四帧重心变化不清楚；
- 焦红 FX 是随机噪点，不构成空间收束；
- 品牌主视觉与运行时图集存在不可接受的质量断层；
- 未提供 Godot 三状态截图，却错误标记为 `APPROVED`。

V1 不得作为回退目标，也不得出现在商店页、README 实机展示或发布候选中。

## 2. V2 已完成改进

- 环境视觉单元升级为 32×32，角色升级为 32×48；
- 保留八个核心语义色，并登记八个只服务材质的过渡色；
- 木地板改为连续纵向木板，不再与墙面砖纹混淆；
- 增加左右墙体、房门、前景遮挡的独立轮廓；
- 床、衣柜、橱柜、沙发、餐桌和关键道具增加材质层次与稳定透视；
- 主角四方向步态加入重心、手臂和脚步差异；
- 惩罚边缘改为低频连续焦蚀轮廓，不再使用全边随机散点；
- 新增 `bedroom_benchmark.png`，只使用实际导出图集完成 640×360 确定性构图。

## 3. 分项质量门

| 资产组 | 当前状态 | 进入 APPROVED 前必须提供 |
|---|---|---|
| 环境 | REVIEW | 五房间第一轮截图；木地板、瓷砖、墙面和出口在原生倍率可区分 |
| 道具 | REVIEW | 关键物在无发光描边时可识别；两轮变体前后对照 |
| 秦峥 | REVIEW | 四方向 8–10 FPS 动图或录屏；脚步事件与着地帧一致 |
| UI | REVIEW | 标题、文本框、时钟、按住确认在 640×360 无溢出 |
| FX | REVIEW | 五个房间的惩罚截图；降低闪烁版本；脚下与最近出口始终可见 |
| 叙事特写 | REVIEW | 与游戏场景同屏截图；确认没有生成图风格突变和责任表达漂移 |
| 品牌概念图 | APPROVED | 已明确标注为概念目标，不冒充实机 |

## 4. 评审顺序

1. 卧室第一轮：检查负空间、主角轮廓、门洞和材质；
2. 厨房第一轮：检查瓷砖、橱柜、收据暖色焦点；
3. 儿童房两轮：检查生活痕迹和儿童画变化可回忆性；
4. 客厅两轮：检查家庭布局、婚纱照与时钟视线分离；
5. 五个房间惩罚态：检查逐灯熄灭、暗角与焦红侵入；
6. UI 与特写叠加：检查视觉风格、中文字级和安全区。

任一阶段发现关键轮廓必须依赖放大、发光描边或文字标签才能识别，则退回 `BLOCKOUT`；不得通过增加提示掩盖美术问题。
